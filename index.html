eMMC Internals

Normal memory

  • Read: reads data
  • Write: writes data

NAND flash memory

  • Read: reads data
  • Write: kind of writes data (1→0)
  • Erase: erases a whole erase-block (0→1)

Erase-blocks have a limited program/erase cycles

Flash Translation Layer

Firmware is responsible for:

  • Wear leveling
  • Bad block management
  • Some of the eMMC bus communication

VTU00M 0xF1 bug

FTL metadata gets corrupted once

⇒ FTL raises a CPU exception on boot

⇒ CPU enters an infinite loop

⇒ eMMC is essentially dead


Samsung's patch: Metadata about to get corrupted? Halt CPU!
